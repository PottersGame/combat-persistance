## 2025-04-05 - URL Injection in API Requests
**Vulnerability:** External HTTP API requests (Mojang API in `SkinManager.java`) concatenated user-controlled strings (usernames and UUIDs) directly into the URL path without URL-encoding. This allows malicious input with special characters (like `/`, `?`, `#`, or `%`) to perform URL injection, altering the endpoint's path or query parameters, potentially causing server errors, accessing unintended endpoints, or bypassing input filters.
**Learning:** Even when the input is assumed to be validated or has length constraints (e.g., username <= 16 chars), it's unsafe to trust user-controlled inputs when constructing URLs. Special characters must be properly encoded.
**Prevention:** Always URL-encode user-controlled identifiers (e.g., Minecraft usernames or UUIDs) using `java.net.URLEncoder.encode(input, StandardCharsets.UTF_8)` before incorporating them into URL strings, especially for external API calls like those to Mojang.
