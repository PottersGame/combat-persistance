## 2024-05-24 - Missing Timeouts and URL Encoding on HTTP Requests
**Vulnerability:** The HTTP requests to Mojang's API in SkinManager.java appended user input directly to the URL without encoding it, leading to possible URL injection or path traversal. Additionally, the initial URL connection lacked connect and read timeouts, and it lacked an explicit HTTP method, creating a Denial of Service (DoS) and potential thread exhaustion risk.
**Learning:** External API integrations that accept user input require defensive practices, including proper URL encoding of variables to prevent manipulation of the URL structure, and timeouts to ensure external failures don't hang internal threads.
**Prevention:** Always URL-encode user-controlled data when appending to URLs using `URLEncoder.encode(input, StandardCharsets.UTF_8)`. Always configure timeouts (e.g., `setConnectTimeout` and `setReadTimeout`) on `HttpURLConnection` instances and explicitly set the request method for clarity and security.
