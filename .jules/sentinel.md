## 2024-04-03 - [SSRF via Unsanitized Minecraft Usernames]
**Vulnerability:** URL Injection / Server-Side Request Forgery (SSRF) risk found in `SkinManager.java` where user-controlled inputs (`identifier` and `uuid`) were being concatenated directly into URLs targeting Mojang's API (`https://api.mojang.com/users/profiles/minecraft/` and `https://sessionserver.mojang.com/session/minecraft/profile/`).
**Learning:** Even when inputs seem constrained (e.g. Minecraft usernames generally restricted to 16 alphanumeric characters), they can still be manipulated (especially by cracked/offline clients or via command arguments like `/skin <malicious_payload>`) to inject query parameters or alter the URL path if passed directly into string concatenation for an HTTP request.
**Prevention:** Always URL-encode user-controlled data (`java.net.URLEncoder.encode(input, StandardCharsets.UTF_8)`) before embedding it into URL strings to ensure it is treated strictly as data and not part of the URI structure.
