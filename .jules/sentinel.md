## 2024-05-15 - Thread Starvation DoS & URL Injection Vulnerabilities
**Vulnerability:** Blocking operations (BCrypt hashing, file I/O, and network requests) were running on the default `ForkJoinPool.commonPool()` which can cause thread starvation and Denial of Service (DoS). Also, user-controlled inputs were concatenated directly into Mojang API URLs in `SkinManager.java`, causing a URL injection/SSRF vulnerability.
**Learning:** The default `ForkJoinPool` is sized based on the number of available processors. Running blocking IO/crypto tasks on it can easily exhaust all threads, causing the entire server to hang or become unresponsive. Unencoded user input in URLs can allow attackers to manipulate requests or trigger SSRF.
**Prevention:** Always provide a dedicated, unbounded executor (like a cached thread pool) for blocking operations in `CompletableFuture` (e.g., `Combatpersistence.IO_EXECUTOR`). Always URL-encode user-controlled identifiers before appending them to URLs using `URLEncoder.encode(..., StandardCharsets.UTF_8)`.
