## 2024-05-24 - DoS and URL Injection vulnerabilities in Mojang API wrapper
**Vulnerability:** External API wrapper missing connection timeouts (`nameConn.setConnectTimeout` and `nameConn.setReadTimeout`) in `SkinManager.java` which could lead to thread exhaustion (Denial of Service). In addition, user input (`identifier` and `uuid`) was concatenated directly into API endpoints without URL encoding, leading to potential URL injection.
**Learning:** This repo interfaces with Mojang API to dynamically resolve skins based on player names and UUIDs. The codebase relies on HTTP connections without appropriate settings that can hang indefinitely if the API doesn't respond or is rate limited, and doesn't sanitize input.
**Prevention:** Always set `connectTimeout` and `readTimeout` on `HttpURLConnection` for external API calls, explicitly set methods via `setRequestMethod`, and always URL encode user-controlled parameters (`URLEncoder.encode(param, StandardCharsets.UTF_8)`) when used in URL construction.
